# Code of Conduct

## Purpose

This project is maintained for serious engineering work around runtime systems, automation, integration and industrial control software. The project welcomes useful technical contributions, precise bug reports, reproducible diagnostics and respectful discussion.

## Expected behavior

Contributors and maintainers are expected to:

```text
- communicate respectfully and professionally
- focus critique on code, architecture, evidence and reproducible behavior
- provide clear context when reporting bugs or proposing changes
- respect project boundaries, licensing and security rules
- avoid publishing secrets, credentials, exploit details or unsafe operational instructions
- help keep discussions useful for maintainers and future contributors
```

## Unacceptable behavior

The following behavior is not acceptable:

```text
- harassment, threats, insults or personal attacks
- discriminatory language or exclusionary behavior
- deliberate disruption of development discussions
- public disclosure of private security vulnerabilities
- publishing real credentials, private keys, customer data or confidential deployment details
- encouraging unsafe operation of industrial systems
- repeated low-effort noise after maintainer guidance was given
```

## Security-sensitive discussions

Security vulnerabilities, credential leaks, unsafe deployment behavior and exploit paths must not be reported through public issues. Use the process in `SECURITY.md`.

## Enforcement

Maintainers may edit, hide, close or delete content that violates this Code of Conduct. Maintainers may also restrict participation by users who repeatedly ignore project rules or create risk for the project, its users or deployed systems.

## Reporting conduct issues

Report conduct issues to the repository maintainer through a private GitHub contact path where possible. Include the relevant links, timestamps and context so the issue can be reviewed fairly.
