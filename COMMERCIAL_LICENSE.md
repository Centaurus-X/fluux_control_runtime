# Commercial License

Commercial licensing is available separately from the GPL-3.0-or-later option.

This file documents the intended commercial licensing path. It does not grant commercial rights by itself. A commercial license requires a separate written agreement with the project owner.

## Purpose

The commercial license option is intended for users or organizations that need rights which are different from, or additional to, the GPL-3.0-or-later open-source option.

Typical commercial use cases include:

```text
- proprietary deployment rights
- closed-source integration rights
- redistribution rights
- customer-specific runtime integration
- industrial automation field deployment
- paid support and maintenance
- warranty, liability or service-level terms
- custom feature development or consulting
```

## Relationship to the GPL option

This repository is intended to use the following dual licensing model:

```text
GPL-3.0-or-later OR Commercial License
```

If no separate written commercial agreement exists, the GPL-3.0-or-later option applies. If a written commercial agreement exists, that agreement defines the rights, obligations, support scope and limitations for the licensed party.

## Commercial license checklist

A final commercial agreement should clearly define:

```text
- licensed repositories and components
- allowed environments and deployment scope
- source-code access and modification rights
- redistribution rights
- support and maintenance responsibilities
- update and patch delivery model
- warranty and liability boundaries
- confidentiality obligations
- security disclosure process
- term, termination and renewal
- pricing or commercial consideration
```

## Important safety note

Industrial automation software must be validated for the target environment before production use. A commercial license does not replace operational testing, safety assessment, security hardening, backup planning, monitoring or acceptance testing.
